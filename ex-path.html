01. ドワーフ・ガンツのカレントディレクトリ

クレアさんよぅ、 HTML ファイルに出てくるこんなやつがいまいち分からないんじゃが？

<a href="./game/tetris.html">

あと CSS ファイルにも似たようなものが登場するじゃろ？ なんなんじゃ、この蟻のウンコみたいなやつは？

h1.top {
  background-image: url("./../images/hero.jpg");
}

「相対パス」のことですね。簡単ですよ。

解説 01

ここをクリックすれば解説が表示されます。

相対パスとな？

相対パスというのは、基準となるファイルが存在しているフォルダやディレクトリから、別のファイルまたはフォルダやディレクトリへの経路のことです。

ん？ ディレクトリとは？

ディレクトリはフォルダと同じです。ファイルたちや自分以外のディレクトリを格納する入れ物のことです。

いやいや、フォルダと同じなら、フォルダと言ってくれ。

Windows などの GUI 環境ではフォルダ、 UNIX や Linux もしくはファイルシステム上の概念としてはディレクトリと呼ぶことが多いです。これからはディレクトリで統一して話を進めます。

すでに話がこんがらがってきた。念のため確認しておくぞ。

ディレクトリ
フォルダのこと。
相対パス
基準となるファイルが存在しているディレクトリから、別のファイルやディレクトリへの経路のこと。

はい。また相対パスの対義語は「絶対パス」です。ルートディレクトリからの経路のことを言います。ちなみにルートディレクトリとは、木構造の最上位に位置するディレクトリのことで、「 / 」単独で表します。

  • / (root)
    • dir1/
      • dir1-1/
        • image.png
    • dir2/
      • dir2-1/
        • file2-1-1.txt
        • file2-1-2.txt
      • file2-2.txt
    • file3.txt
    • index.html

また一度に色々出てきたぞ。まとめるとこんな感じか？

ルートディレクトリ
木構造の最上位に位置するディレクトリのこと。
絶対パス
ルートディレクトリから別のファイルやディレクトリへの経路のこと。

そうですね。例えば https://example.com/index.html というウェブページがあった場合、ドメイン名 example.com のすぐ後ろの / がこのサイトにおけるルートディレクトリを表します。絶対パスは先頭に / が付きます。このサイトにおいては、ファイル index.html の絶対パスは /index.html なわけです。

で、経路とは何のことじゃ？

例えばこんなディレクトリ構成があったとします。ちなみにディレクトリは、末尾に / を付けます。省略することも可能です。この練習帳では省略しません。

  • / (root)
    • dir1/
      • dir1-1/
        • image.png
    • dir2/
      • dir2-1/
        • file2-1-1.txt
        • file2-1-2.txt
      • file2-2.txt
    • file3.txt
    • index.html

で、いま index.html から image.png を参照するにはどんな経路をたどればいいでしょう？

え？ えっと……こ、こんな感じか？

ふざけてるんですか？ 教えるのやめますよ。だいたい何なんですか、そのアイコンは？

すまんすまん。じゃが、分からんのじゃ。

たとえ分からなくてもそうはならないでしょ？

すまんて。……じゃあ、こんな感じか？

さすがですね。

お！ さすがワシじゃ。正解か？

いえ、いい感じに間違ってます。答えはこうです。

いやいやいやいや、クレアさんよう。そいつは話が違う。 index.html から image.png を参照したいんじゃろ？ 出発点からしておかしいわ。頂点からたどるのは相対パスではなくて、絶対パスのはずじゃ。

フッ、注意力散漫ですね。私はさっき相対パスについて「基準となるファイルが存在しているディレクトリから、別のファイルやディレクトリへの経路のこと」と言いました。「基準となるファイルが存在しているディレクトリから」と言ったのであって、「基準となるファイルから」と言ったのではありません。今回、頂点からたどっているのは、たまたま index.html が存在しているディレクトリがルートディレクトリだったからに過ぎません。

なんじゃ、その一休さんの頓智みたいな話は……。

相対パスを辿るには次のような原則があります。

原則 1
起点は基準ファイルが存在するディレクトリである。基準ファイルではない。またこのディレクトリを「カレントディレクトリ」と呼び ./ で表す。
原則 2
相対パスは親・または子の関係のみを辿る。兄弟関係を直接辿ったり、一足跳びに祖先や子孫に到達することはできない。

つまりワシは出発点で間違って（原則 1 に違反）、出発点から兄弟関係にある dir1 に行ったことでさらに間違った（原則 2 に違反）ということじゃな？

そうです。だからさすがだと言ったのです。

むむむ。……おっ、じゃが、ワシは index.html と同じ階層にあるファイルを参照する、こんな感じのハイパーリンクをみたことがあるぞ！ 

<a href="menu.html">

これは index.html と同じ階層にある menu.html を参照しているんじゃから、兄弟関係じゃないのか？

  • / (root)
    • images/
      • image.png
    • css/
      • normalize.css
      • style.css
    • js/
      • main.js
      • game.js
    • menu.html
    • index.html

そ、それは……

お？ クレアさんも間違うことがあるんじゃな？

むむむ……。

まあ、人間誰でも間違うことはあるもんじゃ、気にせんでええぞ。へへへっ！

ふふっ、なんちゃって。……そんなことですぐ喜んじゃうなんて可愛いですね。カレントディレクトリを表す ./ は省略することができます。だからハイパーリンクに書かれた相対パスの menu.html は本当は ./menu.html なんです。

……ワシをいじめて楽しいか？

カレントディレクトリ配下のファイルを参照するには、直接そのファイル名を指定するか、そのファイル名の先頭に ./ を付けて指定することができます。あと一応、厳密に言えば「カレントディレクトリ」というのは、コマンドプロンプトなどを使っている際の作業ディレクトリのことを指します。

02. ドワーフ・ガンツの子孫パス

カレントディレクトリ配下のファイルの参照の仕方は分かった。相対パスは親もしくは子関係を辿るという話じゃったな。まずは、子関係を辿るにはどうすればいいんじゃ？

親関係を辿る方じゃなくて、子関係の方から攻めますか？

なんとなく、そっちの方が簡単そうじゃからな、へへへ。

なかなかよい嗅覚ですね。

解説 02

ここをクリックすれば解説が表示されます。

子関係を辿るのはとても簡単です。カレントディレクトリを出発点として、ディレクトリを順に追記していくだけです。例えばこんなディレクトリ構成で考えてみます。

  • / (root)
    • public_html/
      • ref/
        • images/
          • coffee.jpg
          • cake.jpg
          • tea.jpg
        • css/
          • styles.css
      • index.html

index.html から coffee.jpg を参照してみましょう。

ワシにやらせてくれ。名誉挽回じゃ。原則 1 によって、起点になるのはカレントディレクトリ ./ じゃったな。この木構造の場合、 ./ は実際には public_html/ を指しているはずじゃ。

そうですね。続けてください。

原則 2 によって「相対パスは親または子の関係のみを辿ることができる」んじゃから、次はカレントディレクトリ ./ の子ディレクトリ ref/ に行く。この時点で ./ref/ じゃ。

順調です。

再び原則 2 を適用して、ディレクトリ ref/ の子ディレクトリ images/ に行く。この時点で ./ref/images/ じゃ。

はい。

もう一度原則 2 を適用すれば、目的のファイル coffee.jpg じゃから、これも追記すれば求める相対パスは ./ref/images/coffee.jpg じゃ。どうじゃ？

完璧ですね。

なんか楽しくなってきたぞ。

単純で子どもみたいですね。一つだけ付け加えておくと、カレントディレクトリは省略できるので ref/images/coffee.jpg でも大丈夫です。

そうじゃったな。

設問 02

index.html から styles.css への相対パスを書いてください。

解答例 02

解答例です。

css/styles.css
<!-- または -->
./css/styles.css

03. ドワーフ・ガンツの祖先パス

カレントディレクトリ配下のファイルの指定の仕方も、子ディレクトリ配下のファイルの指定の仕方も分かった。あとは祖先ディレクトリの指定の仕方じゃな。

そうですね。これが分かれば、相対パスの指定はもう何でもできちゃいます。

解説 03

ここをクリックすれば解説が表示されます。

親関係を辿る場合、子関係と違ってひとつ重要なことがあります。子関係と親関係では何が違うか分かりますか？

え？ 子関係は何と言うか、下への関係というか、それに対して親関係と言うのは上への関係というか、そんな感じか？

はぁ……。

違うのか？

まあ、別に違いはしないのですが、……子要素というのは 0 個以上存在するのに対して、親ディレクトリと言うのは常に 1 個しか存在しません。

いや、ルートディレクトリに親関係はないじゃろ？

よいツッコミですね。ルートディレクトリは親が存在しない例外的な唯一のディレクトリです。

で、親が 1 個しか存在しないというのは、何か重要なのか？

とても重要です。では、こんなディレクトリ構成で考えてみましょう。 tamago1.html から kuwagata.html を参照したいとします。

  • / (root)
    • cho/
      • sanagi/
        • aomusi/
          • tamago1.html
          • tamago2.html
          • tamago3.html
    • kuwagata.html

いや、蝶は蛹の親じゃないし、蛹も青虫の親じゃないじゃろ？

いいんです、そんなことは！

tamago1.html から kuwagata.html を辿ればいいんじゃな？ 原則 1 によって起点はカレントディレクトリ ./ じゃから、実際には aomusi/ ディレクトリじゃな。

はい。

原則 2 によって「相対パスは親・または子の関係のみを辿る」んじゃから、次は sanagi/ じゃ。カレントディレクトリに続けて ./sanagi/ じゃろ？

残念ながら違います。それでは、カレントディレクトリ配下の sanagi/ ディレクトリを意味してしまいます。

そ、そう言えばそうじゃな。でもカレントディレクトリ配下に sanagi/ ディレクトリはないんじゃから、ないということに免じて親ディレクトリの方に行ってくれたりはしないんかな？

そんなことに免じられません。カレントディレクトリ配下の sanagi/ ディレクトリを探しに行って、見つからなければそこで終わりです。

じゃあ、どうすればいいんじゃ？

まあ、これは知らないとどうにもなりませんね。親ディレクトリは ../ で表します。

でもそれだと、どのディレクトリか……いや、そうか、親ディレクトリは常に 1 つじゃったな。なんか名探偵みたいじゃな。……じゃあ ./sanagi/ じゃなくて ./../ なんじゃな？

そうです。続けてみてください。

sanagi/ ディレクトリのさらに親ディレクトリを辿るから、次はどうなるんじゃ？ カレントディレクトリが ./ で、親ディレクトリが ../ じゃったから、祖父ディレクトリは .../ か？

面白いですね。天才ですか？

お？ じゃあ、合ってたんじゃな？

いえ、違いますけどね。原則 2 をよく思い出してください。「相対パスは親・または子の関係のみを辿る」んです。いきなり祖父ディレクトリには辿り着けません。それに .../ なんて記号は存在しませんし。

じゃあ、……カレントディレクトリの親の親じゃから ./../../ になるのか？

そうです。

よし。もう分かったぞ。 ./../../ は実際には cho/ ディレクトリを指している。もう一つ上に行く必要があるから ./../../../ じゃな？ これが実質的には、ルートディレクトリ / を指していて kuwagata.html はその直下じゃから、求める相対パスは ./../../../kuwagata.html じゃな？

はい。

この場合もカレントディレクトリを省略して ../../../kuwagata.html と書けるんかな？

書けます。ばっちりです。

styles.css から hero.png への相対パスを書いてください。

04. ドワーフ・ガンツの応用

これで相対パスについては、すべて学んだことになるのかな？

はい。

相対パスを使って、他のサイトへの参照を貼ったりはできないのかな？

できません。最初に示した 2 つの原則で到達できない対象は相対パスで表記できません。

解説 04

ここをクリックすれば解説が表示されます。

念のため、最初に学んだ 2 つの原則を復習しておくと、こんな感じじゃったな？

原則 1
起点は基準ファイルが存在するディレクトリである。基準ファイルではない。またこのディレクトリを「カレントディレクトリ」と呼び ./ で表す。
原則 2
相対パスは親・または子の関係のみを辿る。兄弟関係を直接辿ったり、一足跳びに祖先や子孫に到達することはできない。

はい。これさえ押さえておけば何も難しいことはありません。念のため、こんなディレクトリ構造で確認してみましょう。

  • / (root)
    • tokyoto/
      • taitoku/
        • asakusa/
          • sensoji.temple
          • nakamise.shop
          • hoppy.street
        • ueno/
          • ueno.park
          • sinobazu.pond
          • saigo.statue
          • ameyoko.market
      • nakanoku/
        • broadway/
          • sanmall.shop
          • sikinomori.park
        • araiyakusi/
          • baisyoin.temple
    • chibaken/
      • urayasu/
        • disney.park

観光気分で、浅草寺（ sensoji.temple ）から出発してみることにしましょう。

どこへ行くんじゃ？

まずは、西郷像（ saigo.statue ）へでも行きましょう。

よっしゃ。起点がカレントディレクトリ（ ./ ）つまり浅草（ asakusa/ ）じゃな。ひとつ親に遡って（ ./../ ）台東区（ taitoku/ ）を経由して、上野（ ./../ueno/ ）へ行くと、西郷像（ ./../ueno/saigo.statue ）に到着じゃ。

楽しそうですね。じゃあ、そこから四季の森公園（ sikinomori.park ）へ行ってみてください。

一気に行くぞ！ カレントディレクトリの ueno/ が起点で 2 回遡って tokyoto/ まで行ってから、後は順に下って行けばいいから、求める相対パスは ./../../nakanoku/broadway/sikinomori.park じゃな。

そこからディズニーランド（ disney.park ）へ行って終わりにしましょう。

一旦ルートディレクトリまで行かなきゃいかんな。カレントディレクトリ（ /broadway ）から 3 回遡ってから下るから、求める相対パスは ./../../../chibaken/urayasu/disney.park じゃな。

完璧ですね。
